## 1. Назначение

Этот пример показывает, как подключить интерпретатор Python к приложению Lazarus через **`TPythonConnector`**. Он позволяет активировать Python, выполнять скрипты, получать вывод, вычислять выражения и обмениваться переменными между Pascal и Python.

Используйте этот sample как первый тест перед работой с более сложными компонентами проекта **CHATGPT**, которые зависят от Python: AI, OpenCV, CNN, YOLO, аудио и машинное обучение.

---

## 2. Что проверяет sample

- выбор Python DLL/SO или исполняемого файла;
- активацию и остановку интерпретатора;
- режим **`pemDLL`**, который загружает `python3.dll`, `python312.dll`, `libpython3.so` или аналог;
- режим **`pemProcess`**, который запускает Python как внешний постоянный процесс;
- выполнение кода через **`ExecString`**;
- получение вывода и ошибок через **`LastOutput`** и **`LastError`**;
- работу с переменными через **`SetVar`** и **`GetVar`**;
- вычисление выражений через **`Eval`**;
- диагностический отчет с архитектурой, версией Python, режимом выполнения и последней ошибкой.

## 4. Требования

- Lazarus / Free Pascal.
- Пакет **`openai_core`** доступен в пути проекта.
- Python **3.8–3.14**.
- Архитектура Python должна совпадать с архитектурой приложения.

| Приложение | Требуемый Python |
|---|---|
| Windows 64-bit | Python 64-bit |
| Windows 32-bit | Python 32-bit |
| Linux 64-bit | `libpython`/`python3` 64-bit |
| Linux ARM/ARM64 | `libpython`/`python3` той же архитектуры |

Для режима DLL/SO в Linux может понадобиться:

```bash
sudo apt install python3 python3-dev libpython3-dev
```

## 6. Использование

1. Выберите Python DLL/SO или исполняемый файл.
2. Для первого теста оставьте включенным **Use External Process**.
3. Нажмите **Activate Python interpreter**.
4. Проверьте журнал операций.
5. Выполните пример скрипта или напишите свой.
6. Проверьте **SetVar**, **GetVar** и **Eval**.

## 7. Быстрый тест

Вставьте в memo:

```python
x = 10
print("Hello from Python")
print("x =", x)
```

После выполнения прочитайте переменную `x` через **GetVar**. Ожидаемое значение:

```text
10
```

Затем выполните выражение:

```python
x + 50
```

Ожидаемый результат:

```text
60
```

### Важно о `SetVar`

**`SetVar` сохраняет значения как строки**. Поэтому, если указать:

```text
Name: y
Value: 10
```

используйте:

```python
int(y) + 50
```

а не:

```python
y + 50
```

## 8. Режимы выполнения

### `pemProcess`

Рекомендуемый режим для начального тестирования. Он лучше изолирует Python от процесса Lazarus и снижает риск падения приложения при конфликте библиотек.

### `pemDLL`

Загружает Python прямо в процесс приложения. Этот режим может быть быстрее для простых вызовов, но требует точного совпадения архитектуры и корректной библиотеки Python.

## 9. Диагностика

При активации sample выводит отчет с:

- операционной системой;
- архитектурой Lazarus;
- режимом выполнения;
- путем к Python;
- загруженной библиотекой или исполняемым файлом;
- версией Python;
- совместимостью архитектуры;
- последним шагом загрузки;
- последней ошибкой.

Сначала проверяйте этот отчет. Чаще всего проблема связана с несовпадением архитектуры, отсутствующей библиотекой или Python вне PATH.

---

## 10. Частые проблемы

| Симптом | Причина | Решение |
|---|---|---|
| `Failed to load python3.dll` | Python не установлен, DLL отсутствует или архитектура не совпадает | Установите Python той же архитектуры |
| `Failed to load libpython` | Нет dev-пакета в Linux | Установите `python3-dev`/`libpython3-dev` |
| `GetVar` не работает в DLL режиме | Не найдены optional-функции Python C API | Попробуйте `pemProcess` |
| `y + 50` не работает после `SetVar` | Значение является строкой | Используйте `int(y) + 50` |
| Нет вывода | В скрипте нет `print` или произошла ошибка | Проверьте `LastError` и log |
| Работает в терминале, но не в Lazarus | PATH в IDE отличается | Укажите абсолютный путь к Python |

---

## 11. Рекомендации

- Начинайте с **`pemProcess`**.
- Используйте **`pemDLL`** только при необходимости прямой интеграции с Python C API.
- Держите Python и Lazarus в одной архитектуре.
- Для распространения храните runtime Python в папке приложения или в `libs`.
- При ошибке копируйте диагностический отчет.
