## 1. الهدف

توضح هذه العينة كيفية دمج مفسّر Python داخل تطبيق Lazarus باستخدام **`TPythonConnector`**. تسمح العينة بتفعيل Python، تنفيذ السكربتات، التقاط المخرجات، تقييم التعابير، وتبادل المتغيرات بين Pascal وPython.

تُستخدم هذه العينة للتحقق من أن Runtime الخاص بـ Python مثبت أو مدمج بشكل صحيح قبل استعمال مكوّنات أكثر تعقيدًا في الذكاء الاصطناعي، الرؤية الحاسوبية، الصوت، أو تعلم الآلة داخل مشروع **CHATGPT**.

---

## 2. ماذا تعرض هذه العينة

- اختيار مكتبة أو ملف تنفيذي لـ Python متوافق مع معمارية التطبيق.
- تفعيل وإيقاف مفسّر Python.
- وضعا تشغيل:
  - **`pemDLL`**: تحميل `python3.dll` أو `python312.dll` أو `libpython3.so` أو ما يعادلها.
  - **`pemProcess`**: تشغيل Python كعملية خارجية مستمرة.
- تنفيذ السكربتات عبر **`ExecString`**.
- التقاط المخرجات والأخطاء عبر **`LastOutput`** و **`LastError`**.
- قراءة وكتابة المتغيرات العامة عبر **`GetVar`** و **`SetVar`**.
- تقييم التعابير ديناميكيًا عبر **`Eval`**.
- تقرير تشخيص يحتوي على المعمارية، الإصدار، وضع التشغيل، المكتبة المحمّلة، وخطوة الفشل.

## 4. المتطلبات

### Lazarus / Free Pascal

- تثبيت Lazarus.
- توفر الحزمة **`openai_core`** في مسار المشروع.
- فتح المشروع من الملف **`python_demo.lpi`**.

### Python

تم ضبط المكوّن لقبول Python من **3.8 إلى 3.14**.

يجب أن تتطابق المعمارية:

| التطبيق المترجم | Python المطلوب |
|---|---|
| Windows 64-bit | Python 64-bit |
| Windows 32-bit | Python 32-bit |
| Linux 64-bit | `libpython`/`python3` 64-bit |
| Linux ARM/ARM64 | `libpython`/`python3` بنفس المعمارية |

في وضع **DLL/SO** قد تحتاج إلى تثبيت مكتبة التطوير الخاصة بـ Python.

مثال على Debian/Ubuntu:

```bash
sudo apt install python3 python3-dev libpython3-dev
```

## 6. طريقة الاستخدام

1. اختر DLL أو SO أو ملفًا تنفيذيًا لـ Python من القائمة.
2. اترك خيار **Use External Process** مفعّلًا في الاختبار الأول.
3. اضغط **Activate Python interpreter**.
4. راقب لوحة السجلات.
5. نفّذ السكربت الافتراضي أو اكتب سكربتًا جديدًا في memo.
6. استخدم **SetVar** و **GetVar** و **Eval** لاختبار تبادل البيانات بين Pascal وPython.

## 7. اختبار سريع مقترح

### السكربت

الصق هذا في memo:

```python
x = 10
print("Hello from Python")
print("x =", x)
```

ثم اضغط زر تنفيذ السكربت.

### قراءة متغير

في حقل اسم المتغير اكتب:

```text
x
```

ثم اضغط **GetVar**.

القيمة المتوقعة:

```text
10
```

### تقييم تعبير

استخدم التعبير التالي:

```python
x + 50
```

النتيجة المتوقعة:

```text
60
```

### ملاحظة مهمة حول `SetVar`

حاليًا، **`SetVar` يخزن القيم كنصوص**. إذا استخدمت:

```text
Name: y
Value: 10
```

فاستخدم في Eval:

```python
int(y) + 50
```

بدلًا من:

```python
y + 50
```

## 8. أوضاع التشغيل

### `pemProcess` — عملية خارجية

هذا هو الوضع الموصى به للبداية.

المزايا:

- يعزل Python عن عملية Lazarus بشكل أفضل.
- يقلل الأعطال الناتجة عن تعارض DLL/SO.
- مناسب كبداية عند استخدام مكتبات ثقيلة مثل TensorFlow أو OpenCV أو Torch أو Keras.
- يستخدم `python.exe` أو `python3.exe` أو `python3` أو أي ملف تنفيذي موجود في النظام.

### `pemDLL` — مكتبة ديناميكية

يقوم بتحميل Python مباشرة داخل عملية التطبيق.

المزايا:

- تكامل مباشر أكثر مع Python C API.
- قد يكون أسرع في الاستدعاءات البسيطة.

تنبيهات:

- يجب أن تتطابق المعمارية تمامًا.
- يجب أن تصدّر المكتبة الدوال الإلزامية من C API.
- أي خطأ في مكتبة Python قد يؤدي إلى إغلاق عملية Lazarus.

## 9. التشخيص

عند تفعيل Python، تطبع العينة تقريرًا في لوحة السجلات يحتوي على:

- نظام التشغيل المكتشف؛
- معمارية Lazarus؛
- وضع التشغيل؛
- المسار المستخدم؛
- المكتبة أو الملف التنفيذي المحمّل؛
- إصدار Python؛
- توافق المعمارية؛
- الدوال الإلزامية الموجودة؛
- آخر خطوة تحميل؛
- آخر خطأ إن وجد.

استخدم هذا التقرير قبل اعتبار أن المكوّن لا يعمل. أغلب المشاكل تكون بسبب اختلاف المعمارية، غياب مكتبة، أو عدم وجود Python في PATH.

## 10. مشاكل شائعة

| العرض | السبب المحتمل | الحل |
|---|---|---|
| `Failed to load python3.dll` | Python غير مثبت أو DLL ناقصة أو المعمارية غير صحيحة | ثبّت Python بنفس معمارية التطبيق |
| `Failed to load libpython` | حزمة التطوير غير مثبتة على Linux | ثبّت `python3-dev`/`libpython3-dev` |
| Python يعمل لكن `GetVar` يفشل | دوال اختيارية من C API غير متاحة في وضع DLL | جرّب `pemProcess` |
| `y + 50` يفشل بعد `SetVar` | `SetVar` يحقن القيمة كنص | استخدم `int(y) + 50` أو عرّف `y = 10` في السكربت |
| لا تظهر مخرجات | السكربت لا يستخدم `print` أو حدث خطأ | افحص `LastError` والسجل |
| يعمل في الطرفية ولا يعمل في Lazarus | PATH داخل IDE مختلف | استخدم المسار الكامل لـ Python |

## 11. أفضل الممارسات

- اختبر أولًا باستخدام **`pemProcess`**.
- استخدم **`pemDLL`** فقط عند الحاجة إلى تكامل مباشر مع C API.
- اجعل Python وLazarus بنفس المعمارية.
- عند التوزيع، ضع Runtime الخاص بـ Python داخل مجلد التطبيق أو داخل مجلد فرعي `libs`.
- انسخ تقرير التشخيص دائمًا عند الإبلاغ عن خطأ.
- تجنب السكربتات الطويلة داخل memo؛ استخدم اختبارات قصيرة ومتدرجة.

قبل تحليل خطأ في مكوّنات IA أكثر تعقيدًا، شغّل هذه العينة للتأكد من أن Python يتم تحميله بشكل صحيح.

## 13. تحسينات مستقبلية مقترحة

- إضافة زر لتحديد `python.exe` أو `python3` أو DLL أو SO أو dylib يدويًا.
- توضيح ما إذا كان العنصر المختار ملفًا تنفيذيًا أو مكتبة.
- تقسيم الواجهة إلى تبويبات: الإعداد، السكربت، المتغيرات، Eval، والتشخيص.

- حفظ آخر إعدادات مستخدمة.
- التحقق من وجود الملف المحدد قبل التفعيل.
- عرض تحذير عند استخدام `SetVar` مع قيمة رقمية لأنها ستعامل كنص.
