# News - 2026-06-16

Hoje realizamos a reformulação completa do demo do scanner, correções de segurança de memória em ponteiros do Lazarus, padronização internacional das pastas de componentes (IA para AI), geração de ícones e atualização dos instaladores automatizados.

## Novidades e Correções do Dia

### 1. Reconstrução do Demo `disk_tree_ai_dataset_demo`
- **Nova Interface Limpa**: Toda a interface antiga foi removida para dar lugar a um layout simplificado e profissional:
  - **deTarget (`TDirectoryEdit`)**: Adicionado controle nativo para seleção visual rápida de volumes (como `C:\`) ou pastas (como `C:\Windows`).
  - **Scan & Stop**: Botões unificados na barra superior para iniciar a varredura assíncrona ou cancelar a thread em execução.
  - **sgFiles (`TStringGrid`)**: Adicionado grid de exibição detalhada com redimensionamento de colunas e seleção de linhas.
- **Exibição de Metadados Completos**: O grid agora popula em tempo real os seguintes campos:
  - **Depth**: Nível do arquivo em relação à raiz da busca.
  - **Absolute Path**: Caminho completo.
  - **File Name**: Nome do arquivo.
  - **Extension**: Extensão.
  - **Properties**: Atributos e permissões do arquivo (Flags de leitura, escrita, oculto, sistema).
  - **Size**: Tamanho real em Bytes.
  - **Owner**: Identificação do dono/proprietário do arquivo.

### 2. Resolução de Proprietário do Arquivo Multiplataforma
- Implementada a função `GetFileOwner` com diretivas de compilação condicional:
  - **Windows**: Utiliza chamadas da API do Windows (`GetFileSecurityA`, `GetSecurityDescriptorOwner` e `LookupAccountSidA`) para buscar o domínio e nome da conta proprietária.
  - **Linux/Unix**: Utiliza chamadas POSIX nativas (`fpstat` e `getpwuid`) para traduzir o UID do proprietário para o nome de usuário do sistema.

### 3. Correção de Vazamento e Memory Bug de Ponteiro
- **FPathList (`TStringList`)**: Corrigido um bug crítico de segurança em `main.pas` onde variáveis de string locais e temporárias eram convertidas para ponteiro e armazenadas no campo `Node.Data` de elementos da TreeView. Como strings locais no Free Pascal são gerenciadas e destruídas ao sair de escopo, os ponteiros ficavam apontando para memória corrompida (causando Access Violations ou lixo). Substituído pelo controle gerenciado `FPathList` que armazena as strings e passa apenas os índices seguros para os nós da árvore.

### 4. Padronização Global de Nomes (IA para AI)
- **Mudança de Prefixo**: Renomeadas todas as pastas com prefixo `IA` para usar o padrão internacional `AI` (ex: `IA Agent` -> `AI Agent`, `IA Schedulle` -> `AI Schedule`, etc.).
- **Substituição de Caminhos**: Executado script de atualização que substituiu todas as referências de caminhos e nomes de busca em arquivos de pacotes (`.lpk`), projetos (`.lpi`), códigos-fonte (`.pas`, `.lpr`), readmes (`.md`) e scripts de build.
- **Nova Categoria**: O componente `TAIDiskTreeScanner` foi movido da paleta `AI Native Vision` para a paleta comum de utilitários `AI Utilities`.

### 5. Geração de Ícones da Paleta do Lazarus
- Atualizado o gerador de recursos `generate_all_icons.py` com suporte para os componentes `TAIPythonRuntime` (ícone `PR`) e `TAIDiskTreeScanner` (ícone `TS`).
- O script gerou os arquivos de recursos `.lrs` correspondentes e aplicou os patches de inicialização (`{$I ..._icon.lrs}`) e dependência de `LResources` automaticamente nas units.

### 6. Atualização dos Instaladores de Componentes
- Os scripts de instalação rápida `install_components.bat` (Windows) e `install_components.sh` (Linux) foram atualizados para incluir o novo pacote de componentes `ai_files.lpk` nas rotas recomendadas e completas.
- O instalador foi executado com sucesso e recompilou a IDE do Lazarus com todos os pacotes integrados.
