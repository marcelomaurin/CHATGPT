# Atualizações do Dia - 27 de Junho de 2026

## 🌐 Organização do Runtime Chromium (CEF)

Hoje realizamos uma estruturação e correção massiva na forma como o projeto lida com as bibliotecas nativas do Chromium/CEF necessárias para o funcionamento do componente `TAIChromiumBrowser`.

### 1. Estruturação Multiplataforma do Runtime
Organizamos as dependências do CEF em subpastas lógicas, separando claramente os binários por sistema operacional e arquitetura em `runtime/chromium`:
- `windows/win32`
- `windows/win64`
- `linux/x64`
- `linux/arm32`
- `linux/arm64`

### 2. Automação de Download e Instalação (CEF 90.5.4)
Detectamos e corrigimos conflitos de versão onde o componente `TChromiumWindow` exigia especificamente a versão `90.5.4.0` da `libcef.dll`. 
- Desenvolvemos scripts em lote completos (`.bat`) que automatizam o processo inteiro: eles fazem o download do pacote correto (`cef_binary_90.5.4+gc6a4331+chromium-90.0.4430.72`), extraem os arquivos compactados (`.tar.bz2`) via 7-Zip e distribuem os recursos perfeitamente pelas subpastas adequadas.
- Scripts incluídos: `download_cef`, `extract_cef`, `install_cef`, `check_cef` e o orquestrador `setup_cef`.
- Foram implementados conjuntos de scripts dedicados tanto para **Windows 32 bits** quanto para **Windows 64 bits**.

### 3. Ajustes no Versionamento (.gitignore)
Blindamos o repositório impedindo que arquivos muito pesados (`.tar`, `.bz2` e o conteúdo extenso de `bin/`) sobrecarregassem os limites do GitHub (100MB), rastreando exclusivamente os novos scripts de automação.

## 🚀 Refatoração Profunda do `TAIChromiumBrowser`

Deixamos de lado as injeções manuais e frágeis de strings JavaScript para adoção de uma arquitetura baseada estritamente na **DOM API Nativa** fornecida pelo próprio Chromium.

### 1. Funcionalidades DOM
O componente agora incorpora funções maduras para interação robusta com páginas web:
- `DOMList`, `DOMCount`, `DOMGetElement`, `DOMGetProperty`
- `DOMSetValue`, `DOMSetProperty`, `DOMPressEnter`, `DOMDispatchEvent`
- Controle nativo de eventos focados como `DOMFocus`, `DOMBlur`, etc.

### 2. Integração Limpa de Eventos
Foram criados novos disparadores para que a interface Lazarus reaja dinamicamente às ações dentro da página:
- `OnLoadURL`, `OnFinishedLoadURL`
- `OnKeyPressDOM`, `OnClickDOM`
- `OnSetFocusDOM`, `OnOutFocusDOM` e o evento central `OnDOMResult`

### 3. Correção de Conflitos e Atualização de Demonstrações
- Resolvemos as colisões semânticas do Lazarus (o conflito entre a propriedade nova `ClassName` do nosso objeto contra o `ClassName` base de `TObject` substituindo por `DOMClassName`).
- Adicionamos regras para `overload` ocultando avisos do compilador nas rotinas de sobreposição como `Click()`.
- O aplicativo exemplo `chromium_capture_demo` foi inteiramente atualizado para utilizar o novo modelo do DOM, abolindo lógicas avulsas e sendo muito mais idiomático e integrado ao Lazarus, agora também referenciado graficamente pelo `README.md` com prints do seu funcionamento.
