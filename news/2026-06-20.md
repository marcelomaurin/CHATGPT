# News - 2026-06-20

Hoje implementamos o novo pacote de banco de dados **AIDBase**, criamos o componente e o exportador de dicionário de dados (com suporte completo para PostgreSQL e SQLite), traduzimos toda a aplicação de demonstração e tabelas de exemplo para o inglês e atualizamos as documentações globais do repositório.

## Novidades e Correções do Dia

### 1. Novo Pacote `openai_aidbase.lpk`
- Criado o pacote de componentes Lazarus `openai_aidbase.lpk` e registrado sob a aba `'AI DBase'` da paleta de componentes.
- Ele depende dos pacotes `LCL`, `FCL`, `ZComponent` (ZeosLib) e `openai_core`.

### 2. Dicionário de Dados e Motores de Banco
- Criada a classe base `TAICustomDBDictionary` que fornece as propriedades e fluxo padrão de carregamento de metadados.
- Implementado o motor de metadados `TAIPostgreSQLDictionary` para leitura completa do dicionário de dados no PostgreSQL (tabelas, colunas, chaves primárias/estrangeiras, índices, views, triggers, sequences e routines).
- Implementado o motor de metadados `TAISQLiteDictionary` para leitura do dicionário de dados em arquivos SQLite (tabelas, colunas, chaves primárias/estrangeiras, índices, views e triggers).
- Adicionados stubs e classes esqueleto de preparação para MySQL, Firebird, SQL Server e Oracle.

### 3. Exportação em Múltiplos Formatos (Inglês)
- Implementada a unit `aidb_dictionary_exporter.pas` com a classe `TAIDBDataDictionaryExporter`.
- Suporte para exportação do dicionário de dados em:
  - **Texto (Plain Text)**
  - **Markdown** (com tabelas e formatação amigável)
  - **JSON**
  - **AI Prompt** (formato otimizado de instrução para modelos de IA)
- Toda a estrutura gerada pelo exportador (tabelas, cabeçalhos, chaves e relacionamentos) é produzida em inglês.

### 4. Aplicação Demo `db_dictionary_demo`
- Criado o aplicativo de demonstração em `pacote/samples/AI DBase/db_dictionary_demo/`.
- Permite conectar visualmente a bancos PostgreSQL e arquivos SQLite (utilizando caminhos relativos na pasta `database/` para o banco local `etiqueta.db`).
- Toda a interface visual (botões, rótulos, mensagens de progresso, status e diálogos) foi traduzida integralmente para o inglês.
- O arquivo SQLite de exemplo `etiqueta.db` foi recriado contendo tabelas e colunas em inglês (`product`, `selected_product` e `address` no lugar de `endereco`).
